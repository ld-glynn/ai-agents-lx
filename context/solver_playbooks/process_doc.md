# Playbook: Process Document

## Purpose
Draft a new or revised process document that the team can adopt.

## Structure

### 1. Process Name & Purpose
What is this process and why does it exist?

### 2. When to Use
What triggers this process? Who initiates it?

### 3. Roles Involved
| Role | Responsibility |
|------|---------------|
| ... | ... |

### 4. Steps
Numbered sequence. Keep it simple — if a step needs sub-steps, it's too complex.

### 5. Inputs & Outputs
What goes in, what comes out?

### 6. Exceptions & Edge Cases
What if something doesn't fit the normal flow?

### 7. Related Problems
Which catalog problems does this process address? (cite IDs)

## Guidelines
- Write for someone doing this for the first time
- Prefer checklists over paragraphs
- Keep under 1 page if possible
- Include "who to ask" for edge cases
